# Repository Guidelines

## Project Structure & Module Organization
- `src/routes/`: SvelteKit routes with `[lang=lang]/` i18n prefix. Both blogs
  share one implementation via `[section=section]` (`posts` = tech, `life`).
  Endpoints: `feed/` (runtime RSS), `og/` (prerendered OG images),
  `sitemap.xml/`, `robots.txt/`.
- `src/lib/components/`: Svelte components (printer shell UI, printed
  elements, post list/content).
- `src/lib/dictionaries/`: i18n strings (`en.ts` is the canonical shape,
  `zh.ts` must match). Keep keys in sync.
- `src/lib/content.ts`: typed access to Velite output (posts, categories).
- `content/`: source content. Tech posts in `content/posts/YYYY-MM-DD title/`
  with `en.md` + `zh.md`; life posts in `content/life-posts/<slug>/`.
  Categories in `content/categories/{posts,life}.yml`.
- `static/`: static assets, copied verbatim. `_headers` (project root) holds
  Cloudflare cache rules. `static/blog/` is generated by Velite (gitignored).
- `assets/icon.svg`: source of truth for the boxed app icons. `pnpm icons`
  (`scripts/generate-icons.mjs`) regenerates `static/{favicon.ico,icon.png,
  icon-192.png,apple-icon.png}` from it; commit the regenerated files.
  `static/logo.svg` and `static/mask-icon.svg` are hand-maintained.
- Root config: `svelte.config.js`, `vite.config.ts`, `velite.config.ts`,
  `wrangler.jsonc`.

## Build, Test, and Development Commands
- `pnpm install`: Install dependencies (pnpm only; no npm/yarn lockfiles).
- `pnpm dev`: Start local dev server (runs Velite in watch mode).
- `pnpm check`: svelte-check type checking. Run before PRs.
- `pnpm build`: Production build; prerenders every page and OG image.
- `pnpm preview`: Build + preview in the Workers runtime (wrangler dev).
- `pnpm deploy`: Build + deploy to Cloudflare Workers.
CI runs check and build on push via `.github/workflows/ci.yml`. Deployment is
handled by Cloudflare Workers Builds (Git integration), which builds and
deploys on push to main.

## Architecture Notes
- The whole site is prerendered (`prerender = true` in the root layout). The
  worker only handles the `Accept-Language` locale redirect
  (`src/hooks.server.ts`), the extensionless `/feed*` routes, and
  `/api/social`, all cached at the edge with the Cloudflare Cache API.
- `/api/social` serves live GitHub/X profile stats for the social hover
  cards (`src/lib/components/SocialHoverCard.svelte`). It layers caches:
  edge Cache API → `SOCIAL_CACHE` KV last-good snapshot (binding in
  `wrangler.jsonc`; the code tolerates a missing binding in local dev) →
  committed fallback `src/lib/social-fallback.json`.
- OG images are rendered at build time with satori + resvg
  (`src/lib/og/image.ts`); CJK glyph subsets and Twemoji are fetched during
  the build, so builds need network access.
- Icons are inline SVG paths in `src/lib/icons.ts` — no icon packages.

## Coding Style & Naming Conventions
- Language: TypeScript (strict). Framework: SvelteKit 2 + Svelte 5 (runes).
- Indentation: 2 spaces; LF line endings (`.editorconfig`).
- File naming: PascalCase for components (`PrinterShell.svelte`), kebab-case
  or camelCase for modules, camelCase for variables.
- Styling: Tailwind CSS 4 (CSS-first config in `src/app.css`).

## Commit & Pull Request Guidelines
- Commits: clear, imperative messages (e.g., "Update featured works").
- i18n: update both `en` and `zh` where applicable (content and dictionaries).
- Content: new tech posts use `content/posts/YYYY-MM-DD title/` with `en.md`,
  `zh.md`, and images alongside.
- Checks: ensure `pnpm check` and `pnpm build` succeed before review.
